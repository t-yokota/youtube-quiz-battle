# Task Management Guide for Kiro Specs

## 基本方針

Kiroのspec機能を使用する際、tasks.mdファイルの適切な管理は他のAIエージェントとの協働において重要です。

## Task Status Management

### チェックボックス形式でのステータス更新

タスクを実行する際は、tasks.mdファイルを直接編集してチェックボックスの状態を更新してください：

```markdown
- [ ] 未着手のタスク
- [-] 進行中のタスク
- [x] 完了したタスク
```

### ステータス更新のタイミング

```
1. タスク開始時: [ ] → [-]
2. タスク完了時: [-] → [x]
3. タスクが未着手に戻る場合: [-] → [ ]
```

### 正確なタスク行の特定

tasks.mdファイル内で正確なタスク行を特定してください：

**例:**

```markdown
- [ ] 2. Tailwind CSS設定の修正と基本レイアウト
  - Tailwind CSS v4設定の修正
  - 基本的な垂直配置レイアウトの構築
  - 縦画面専用レスポンシブレイアウトの実装
  - _Requirements: 2.1, 2.2_
```

この場合、メインタスクの行（`- [ ] 2. Tailwind CSS設定の修正と基本レイアウト`）のチェックボックスを更新します。

### サブタスクの管理

サブタスクがある場合は、以下の順序で実行してください：

1. **親タスクを"[-]"に設定**
2. **各サブタスクを順次実行**（サブタスクも個別にステータス管理可能）
3. **全サブタスク完了後、親タスクを"[x]"に設定**

## Task Execution Rules

### 1タスク1実行の原則

- **1回の実行で1つのタスクのみ**を完了させる
- タスク完了後は自動的に次のタスクに進まない
- ユーザーの確認を待ってから次のタスクを開始する

### タスク実行前の確認事項

タスクを実行する前に以下を確認してください：

1. **要件ドキュメント**: `requirements.md`の該当要件を確認
2. **設計ドキュメント**: `design.md`の関連設計を確認
3. **依存関係**: 前提となるタスクが完了しているか確認

### タスク完了の判定基準

タスクを"[x]"にする前に以下を確認してください：

- [ ] タスクの説明に記載された全ての項目が実装済み
- [ ] 要件（\_Requirements）が満たされている
- [ ] 動作確認が完了している
- [ ] 必要に応じてテストが実装されている

## File Management

### tasks.mdファイルの直接編集

kiro以外のAIエージェントは、tasks.mdファイルを直接編集してタスクステータスを更新します：

```markdown
# 変更前

- [ ] 2. Tailwind CSS設定の修正と基本レイアウト

# 作業開始時

- [-] 2. Tailwind CSS設定の修正と基本レイアウト

# 作業完了時

- [x] 2. Tailwind CSS設定の修正と基本レイアウト
```

### ファイル編集時の注意事項

- **チェックボックス部分のみ**を変更する
- **タスク名や説明は変更しない**
- **インデントや書式を保持する**
- **\_Requirements部分は変更しない**

## Error Handling

### タスクが見つからない場合

該当するタスクが見つからない場合：

1. tasks.mdファイル全体を確認
2. タスク番号や名前の正確性を確認
3. Phase構成を確認
4. 必要に応じてユーザーに確認を求める

### タスクの修正が必要な場合

タスクの内容や構造を修正する必要がある場合：

1. **ユーザーに確認を求める**
2. 承認後にtasks.mdファイルを更新
3. 関連するdesign.mdとの整合性を保つ

## Collaboration Guidelines

### 他のエージェントとの協働

- タスクステータスは**リアルタイムで更新**する
- 作業中のタスクは"[-]"で明示する
- 完了したタスクは必ず"[x]"に更新する

### 引き継ぎ時の注意事項

他のエージェントにタスクを引き継ぐ際：

1. 現在のタスクステータスを正確に更新
2. 作業内容と進捗状況を明確に記録
3. 次に実行すべきタスクを明示

## Best Practices

### タスク実行の流れ

```
1. タスクの詳細を確認
   ↓
2. 要件・設計ドキュメントを確認
   ↓
3. tasks.md編集: [ ] → [-]
   ↓
4. 実装作業
   ↓
5. 動作確認・テスト
   ↓
6. tasks.md編集: [-] → [x]
   ↓
7. ユーザーに完了報告
```

### 品質保証

- 実装前に設計を理解する
- 実装後に動作確認を行う
- 要件を満たしているか検証する
- 必要に応じてリファクタリングを行う

## Special Notes for Spec Agents

### taskStatusツールについて

**spec専用エージェント**であるkiroは、taskStatusツールを使用してタスクステータスを更新できます：

```
taskStatus(
  taskFilePath: ".kiro/specs/youtube-quiz-battle/tasks.md",
  task: "2. Tailwind CSS設定の修正と基本レイアウト",
  status: "in_progress" | "completed" | "not_started"
)
```

このツールは一般的なAIエージェントでは利用できないため、他のエージェントは直接ファイル編集を行います。

## 注意事項

- **tasks.mdファイルを大幅に編集する場合は事前にユーザーの承認を得る**
- **タスクの順序や依存関係を変更する場合は慎重に検討する**
- **Phase構成を変更する場合はdesign.mdとの整合性を保つ**
- **大幅な変更の場合は必ずユーザーと相談する**

このガイドラインに従うことで、複数のAIエージェント間でのタスク管理が円滑に行われ、プロジェクトの進行が効率化されます。
