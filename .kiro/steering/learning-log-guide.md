# Learning Log まとめガイド

## 基本方針

ユーザーから学習内容の整理を依頼された場合、`learning-notes/learning-log.md` ファイルに内容を追記・更新する。

## Learning Log ファイル管理ルール

### ファイル分類と選択基準

#### 1. 統合型 Learning Log

- **ファイル名**: `learning-notes/learning-log.md`
- **用途**: 複数分野にわたる学習内容の集約
- **使用条件**: 各分野の学習量が少ない、全体把握したい場合

#### 2. 分野別 Learning Log

- **ファイル名**: `learning-notes/{分野名}-learning-log.md`
- **例**: `vue-learning-log.md`, `css-learning-log.md`
- **用途**: 特定分野の学習内容が豊富な場合
- **使用条件**: その分野で継続的に学習を進めている場合

#### 3. 詳細トピック別ドキュメント

- **ファイル名**: `learning-notes/{トピック名}.md`
- **例**: `vue-scoped-styles.md`, `tailwind-setup.md`
- **用途**: 特定トピックの詳細な学習記録
- **使用条件**: 深い学習内容、多くのコード例がある場合

### ドキュメント構造

#### 必須要素

1. **タイトル** - `# Learning Log`
2. **目次** - 学習分野のリンク付き一覧
3. **分野別セクション** - トピックごとに整理
4. **今後の学習予定** - チェックボックス形式

#### 推奨構造例

```markdown
# Learning Log

## 目次

- [Vue.js](#vuejs)
- [CSS/スタイリング](#cssスタイリング)
- [JavaScript](#javascript)

## Vue.js

### Vue Scoped Styles

説明文 + 要点まとめ + コード例

### Vue コンポーネント設計

説明文 + 学習内容

## CSS/スタイリング

### Tailwind CSS

学習内容

---

## 今後の学習予定

- [ ] 関連トピック1
- [ ] 関連トピック2
```

### 内容作成ガイドライン

#### 文章スタイル

- **理解促進重視** - 箇条書きだけでなく、概念や仕組みを理解できる説明文を含める
- **簡潔性とのバランス** - 長すぎない範囲で、理解に必要な説明を提供
- **要点整理** - 学習した重要ポイントを明確に記載
- **実例付き** - 必要に応じてコード例や具体例を含める
- **文脈説明** - なぜそうなるのか、どう動作するのかの背景を簡潔に説明

#### 構成原則

- **分野別整理** - 技術分野ごとにセクション分け
- **段階的蓄積** - 新しい学習内容を既存セクションに追記
- **実用性** - 後で振り返りやすい形で整理

#### 技術的内容

- **概念説明** - 技術的な仕組みや動作原理を理解しやすい文章で説明
- **要点まとめ** - 学習した内容の核心部分を簡潔に記載
- **コード例** - 重要な概念は簡単なコード例で補強
- **変換例** - Before/After形式で変化や処理結果を明示
- **参照リンク** - 詳細な個別ドキュメントがある場合はリンクを設置
- **実用的な文脈** - なぜ重要なのか、どんな場面で使うのかを説明

### 更新・拡張ルール

#### 既存内容の更新

- 新しい学習内容は適切な分野セクションに追加
- 目次の更新を忘れずに行う
- 関連する既存内容がある場合は統合または参照追加

#### 新分野の追加

- 新しい技術分野の場合は新セクションを作成
- 目次に新分野を追加
- 適切な順序で配置（基礎的なものから応用へ）

## 実行手順

1. **内容確認** - ユーザーの依頼内容を整理
2. **ファイル選択** - 統合型/分野別/詳細ドキュメントのどれが適切かを判断
3. **分野判定** - 既存分野への追記か新分野作成かを判断
4. **構造設計** - 追記する場所と内容構成を決定
5. **内容作成** - ガイドラインに従って内容を作成・追記
6. **目次更新** - 必要に応じて目次を更新

## 注意事項

- learning-logは学習内容の集約ファイルとして機能
- 分野別learning-logは特定分野が充実してきた場合に作成
- 詳細な技術解説は個別ファイル（例：vue-scoped-styles.md）で管理
- learning-logでは要点と概要に留める
- 日付記録は不要（内容ベースの整理）
- ユーザーが明示的に分野別ログを指定した場合はそれに従う
- **重要**: 単純な箇条書きの羅列ではなく、理解を深める説明文を心がける
