# Task Execution Guide

## 基本方針

このプロジェクトでは学習を重視したタスク実行を行う。

### 役割分担

**ユーザー（あなた）の役割:**
- すべてのコマンド実行
- 実行結果のフィードバック
- 実装内容の理解と質問
- 学習プロセスの主導

**AI（私）の役割:**
- 実装の設計と実行
- ガイダンスとサポート
- コードの説明と解説
- 次のステップの提案

### タスク実行フロー

1. **タスク開始時**
   - 私がタスクの概要と実装方針を説明
   - 必要なコマンドを提案（実行はあなたが担当）
   - 実装前の設計について議論

2. **実装フェーズ**
   - 私がコードを実装
   - あなたが内容を確認・理解
   - 疑問点があれば随時質問・議論

3. **検証フェーズ**
   - あなたがテストコマンドを実行
   - 結果を私にフィードバック
   - 必要に応じて修正・改善

4. **学習フェーズ**
   - 実装内容の解説
   - 設計判断の理由説明
   - 次のタスクへの準備

### 重要なポイント

- **コマンド実行**: 必ずあなたが実行し、結果を共有
- **理解優先**: 急がず、理解できるまで質問する
- **学習重視**: なぜそう実装したかの理由も重要
- **協働作業**: 一緒に考えながら進める

### 質問の推奨

- 「なぜこの実装方法を選んだのですか？」
- 「この部分の動作原理を教えてください」
- 「他の実装方法との違いは何ですか？」
- 「このコードの意図を説明してください」

このアプローチで、実装スキルと理解の両方を深めていく。
