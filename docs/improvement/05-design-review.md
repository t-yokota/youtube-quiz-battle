## 1. 現行デザインのレビュー

### 総評

機能の骨格（垂直レイアウト・状態に応じた表示切替・44px タッチターゲット）は適切に実装されている。一方で視覚デザインは「Tailwind 既定パレットのプロトタイプ」段階にあり、**「早押しクイズの緊張感・ステージ感」というプロダクト固有の体験が画面から伝わらない**。

### 指摘事項

| # | 指摘 | 重大度 |
|---|---|---|
| V-1 | **アイデンティティ不在**: blue-600 ヘッダー + 白カード + gray 背景は管理画面の文法。クイズ番組的な高揚感がない。色がセマンティック（正解=緑、不正解=赤）以前に、ブランドカラー自体が未定義 | 高 |
| V-2 | **早押しボタンが「ボタン」に見えない**: 単色の角丸長方形（3:4）。早押しクイズ文化の中心であるボタンチェック演出の主役なのに、物理ボタン（円形・台座・LED）のメタファーがない。`docs/assets/button.webp`（実物参考画像）が活かされていない | 高 |
| V-3 | **緊張感の演出不足**: 解答カウントダウンがテキスト「残り N秒」のみ。残り時間の視覚化（リング/バー）、終盤の色変化・脈動がない | 高 |
| V-4 | **状態変化が地味**: QUESTIONING（早押し可能 = 最重要の瞬間）とTALKING の画面差がボタン色と入力欄活性だけ。「今押せる」ことが周辺視野で分からない | 中 |
| V-5 | **正解/不正解のフィードバックが弱い**: 小さなバッジが answer-meta 行に出るのみ。報酬感・ペナルティ感がない | 中 |
| V-6 | **階層が平坦**: ヘッダー / GameInfo / 解答エリアがほぼ同ウェイト。スコアは `○: 2 ×: 1` のテキストで、進行（第N問/全M問）の全体感もない | 中 |
| V-7 | **タイポグラフィ未設計**: システムフォントのみ、数字が等幅でないためタイマー・スコアが桁替わりでガタつく（tabular-nums 未指定） | 中 |
| V-8 | **インタラクション状態の設計不足**: hover はあるが pressed/focus-visible の一貫した設計がない。reduced-motion 配慮なし。answer-result に aria-live がなく SR に伝わらない | 中 |
| V-9 | **FINISHED 画面に演出がない**: 完走の達成感（スコア強調・出題別タイムライン）が薄い | 低 |
| V-10 | **スタイル実装の負債**: 16 進カラー直書きが 8 ファイルに分散、media query コピペ（→ R-8 / Phase D-1 で解消） | - |

## 2. 改善の方向性（スタイルディレクション）

**選定: 「TV クイズ番組のステージ / スコアボード」**

- 根拠: プロダクトの核は「動画出演者との疑似対戦」。視聴者が知っている早押しクイズ番組の視覚言語（暗いスタジオ、スポットライト、電光スコアボード、赤い物理ボタン）をそのまま借りられる。汎用 SaaS 風から最も遠く、かつ説明不要
- 背景: ダークネイビーのステージ（動画の黒枠とも馴染み、没入感が出る。明所のスマホ利用でも動画視聴 UI はダーク基調が標準的）
- アクセント: シグナルレッド（ボタン）+ アンバー/ゴールド（スコア・正解・スポットライト）
- タイポグラフィ: 見出し・数字は太いゴシック + `font-variant-numeric: tabular-nums`。スコアボードの数字はレタースペースを締める
- モーション: 「押せる瞬間」と「結果」にのみ集中投下（QUESTIONING のパルス、正解/不正解バナー、タイマーリング）。それ以外は静的に保つ

## 3. ケース 1: 構成維持・CSS 中心の改善（wireframe-v2-case1.html）

現行のワイヤーフレーム構成（ヘッダー → 動画 → GameInfo → 解答エリア → 早押しボタン、+ ダイアログ）と DOM 役割を維持したまま、見た目と状態演出を刷新。**実装時の変更は各 SFC の `<style>` とテンプレートのクラスのみで収まる**ことを意図している。

主な変更点:
1. ステージ背景（ダークネイビー + ボタン背後の放射スポットライト）
2. GameInfo をスコアボード化: 「Q 03/05」のセグメント風進行表示 + ○× を点灯チップ列で表示。チップ列は**直近5問のスライディングウィンドウ + 三角ページャ**（右端は現在の問題まで）とし、問題数が増えても破綻しない。○/×/スキップ/無解答の 4 パターンを外周円ごと一体の SVG シンボルで描画（サブピクセルずれ防止・フォント非依存）
3. 早押しボタンを物理ボタン化: **真上から見た円形キャップ + 同心円台座** + 押下で沈む（縮小 + 内側影 + 減光）+ RELEASED で LED リングがグロー、QUESTIONING 中は外周パルスリング
4. 解答エリア: タイマーは 2 案を用意 — **①conic-gradient リング / ②解答エリアの枠メーター**（いずれも 12 時起点で時計回りに減少、≤3 秒で赤 + 脈動。実装時にユーザーが選択）。正解/不正解はエリア全体を縁取りフラッシュするバナー表示
5. ガイドテキストは文言のみを上下中央配置（絵文字アイコンなし）。READY ではボタンへ視線誘導する矢印モーション
6. FINISHED はフルスクリーンのリザルトステージ（大型スコア + 問題別タイムライン + もう一度プレイ、細スクロールバー）。当初ケース2のみの提案だったが、採用決定に伴いケース1へ取り込み済み
7. アクセシビリティ: `prefers-reduced-motion` で全アニメーション停止、`:focus-visible` リング統一、answer-result に `aria-live="polite"`（実装時要件としてコメント記載）

## 4. ケース 2: ゲーム体験向上のための構造変更案（wireframe-v2-case2.html）— 不採用（2026-06-12・アーカイブ）

構成自体を見直し、**「押す体験」をレイアウトの主役にする**抜本案。

### 変更の論点

| 論点 | 現行 | ケース 2 |
|---|---|---|
| ボタン位置 | 解答エリア下の余白に浮く | **画面最下部のサムゾーンに固定**（片手持ちの親指が最短距離で届く）。ボタンはステージ床から生える台座つき |
| 解答入力 | 常設の入力欄（非活性表示が大半の時間を占める） | **ANSWERING 時のみボトムシートがスライドイン**。ソフトキーボードと同じ方向から出るため、キーボード出現と視覚的に連続する（実装時は visualViewport 連動） |
| 進行情報 | GameInfo 行 | 動画直下の**スリム HUD**（問題ドット + ○×チップ + 状態ラベル）に圧縮し、垂直スペースをボタンに譲る |
| 結果表示 | answer-meta 内のバッジ | **画面中央のリザルトバナー**（正解=ゴールド/不正解=レッド）を 1.2 秒オーバーレイ |
| FINISHED | 同一レイアウト内で差し替え | **フルスクリーンのリザルトステージ**: 大型スコア + 問題別タイムライン（○×/スキップ）+ もう一度プレイ |
| 待機状態 | ボタンがグレー化 | ボタンが**台座に沈み暗転**+ HUD に状態ラベル（「他プレイヤー解答中」等）。押せる瞬間はスポットライト点灯 + パルスで復帰 |

### 採否判断の材料（実装コスト）

- ボトムシート化は GamePanel の再構成 + visualViewport 連動が必要（2〜3 タスク規模）
- HUD 化は GameInfo の改修のみ（小）
- リザルトステージは FINISHED 分岐の差し替え（中）
- **段階導入可**: HUD + ボタン配置（小〜中）→ リザルトステージ（中）→ ボトムシート（大）の順に切り出せる
