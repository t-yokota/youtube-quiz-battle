# Requirements Document

## Introduction

YouTube Quiz Battleは、YouTubeに投稿されたクイズ動画を視聴しながら、動画内で出題された問題にリアルタイムで早押し解答し、動画出演者と疑似的にクイズ対戦を楽しむWebアプリケーションです。

プレイヤーは動画の進行に合わせて早押しボタンを押し、制限時間内に解答を入力することで、インタラクティブなクイズ体験を得ることができます。スマートフォンブラウザでの利用を主目的とし、直感的で没入感のあるゲーム体験を提供します。

## Requirements

### Requirement 1

**User Story:** プレイヤーとして、YouTubeクイズ動画を視聴しながらリアルタイムで早押し解答したい。そうすることで、受動的な動画視聴ではなく、能動的でインタラクティブなクイズ体験を楽しめる。

#### Acceptance Criteria

1. WHEN プレイヤーがWebページにアクセス THEN システムはYouTube動画プレイヤーと早押しボタンを表示する
2. WHEN 動画内でクイズ問題が出題される THEN システムは早押しボタンを有効化し、プレイヤーが押下可能な状態にする
3. WHEN プレイヤーが早押しボタンを押下 THEN システムは動画を一時停止し、解答入力フォームを表示する
4. WHEN プレイヤーが解答を入力・送信 THEN システムは正誤判定を行い、結果を表示して動画を再開する
5. WHEN 動画内の全ての問題が終了 THEN システムは最終スコアを表示する

### Requirement 2

**User Story:** プレイヤーとして、スマートフォンで快適にクイズゲームをプレイしたい。そうすることで、いつでもどこでも手軽にクイズを楽しめる。

#### Acceptance Criteria

1. WHEN プレイヤーがスマートフォンでアクセス THEN システムは縦画面専用のレスポンシブUIを表示する
2. WHEN プレイヤーが横画面に回転 THEN システムは縦画面への回転を促す警告を表示する
3. WHEN プレイヤーが早押しボタンをタッチ THEN システムは即座に反応し、適切な視覚・音声フィードバックを提供する
4. WHEN プレイヤーが解答を入力 THEN システムはソフトキーボードに対応した画面調整を行う
5. WHEN プレイヤーがPCでアクセス THEN システムはスペースキーでの早押し操作にも対応する

### Requirement 3

**User Story:** プレイヤーとして、公正なクイズゲームをプレイしたい。そうすることで、動画を先読みすることなく、リアルタイムでの解答体験を楽しめる。

#### Acceptance Criteria

1. WHEN プレイヤーが動画のシークバーを操作 THEN システムはシーク操作を検出し、適切な制御を行う
2. IF シーク無効設定が有効 THEN システムは動画時間を元の位置に強制リセットする
3. IF シーク無効設定が無効 THEN システムはゲーム状態を待機状態に固定し、次の問題から参加可能にする
4. WHEN プレイヤーが連続視聴している THEN システムは正常な状態遷移を継続する
5. WHEN プレイヤーが解答制限時間を超過 THEN システムは自動的に解答を締め切り、次の状態に遷移する

### Requirement 4

**User Story:** プレイヤーとして、多様な解答形式に対応したクイズを楽しみたい。そうすることで、表記揺れを気にせずに解答に集中できる。

#### Acceptance Criteria

1. WHEN プレイヤーが解答を入力 THEN システムは大文字小文字の違いを無視して判定する
2. WHEN プレイヤーが全角・半角文字を混在して入力 THEN システムは文字幅を統一して判定する
3. WHEN プレイヤーがカタカナ・ひらがなを使い分けて入力 THEN システムは仮名表記を統一して判定する
4. WHEN プレイヤーが前後に空白を含めて入力 THEN システムは空白を除去して判定する
5. WHEN 問題に複数の正解パターンが設定されている THEN システムはいずれかに一致すれば正解と判定する

### Requirement 5

**User Story:** プレイヤーとして、ゲーム進行状況を把握しながらプレイしたい。そうすることで、現在の状況を理解し、適切なタイミングで操作できる。

#### Acceptance Criteria

1. WHEN ゲームが進行中 THEN システムは現在の問題番号を表示する
2. WHEN プレイヤーが解答権を獲得 THEN システムは残り解答回数と制限時間を表示する
3. WHEN プレイヤーが解答を送信 THEN システムは正誤判定結果を即座に表示する
4. WHEN ゲームが進行中 THEN システムは累計の正解数・不正解数を表示する
5. WHEN システムがリソースを読み込み中 THEN システムは読み込み状況を示すローディング画面を表示する

### Requirement 6

**User Story:** プレイヤーとして、音声フィードバックのあるクイズゲームを楽しみたい。そうすることで、より没入感のあるゲーム体験を得られる。

#### Acceptance Criteria

1. WHEN プレイヤーが早押しボタンを押下 THEN システムはボタン押下音を再生する
2. WHEN プレイヤーの解答が正解 THEN システムは正解音を再生する
3. WHEN プレイヤーの解答が不正解 THEN システムは不正解音を再生する
4. WHEN プレイヤーが音声設定を変更 THEN システムは音声の有効/無効を切り替える
5. WHEN 複数の音声が重複再生される THEN システムは前の音声を停止して新しい音声を再生する

### Requirement 7

**User Story:** プレイヤーとして、エラーが発生した際に適切な対応ができるようにしたい。そうすることで、問題が発生してもゲームを継続または再開できる。

#### Acceptance Criteria

1. WHEN YouTube動画の読み込みに失敗 THEN システムはエラーダイアログを表示し、再読み込みを促す
2. WHEN 音声ファイルの読み込みに失敗 THEN システムはエラーダイアログを表示し、再読み込みを促す
3. WHEN クイズデータの読み込みに失敗 THEN システムはエラーダイアログを表示し、再読み込みを促す
4. WHEN 予期しないエラーが発生 THEN システムは汎用エラーメッセージを表示し、復旧手順を案内する
5. WHEN エラーダイアログが表示される THEN プレイヤーは「再読み込み」ボタンでページをリロードできる

### Requirement 8

**User Story:** 開発者として、クイズデータを簡単に追加・管理したい。そうすることで、新しいクイズ動画に対応したデータを効率的に作成できる。

#### Acceptance Criteria

1. WHEN 開発者がクイズデータを作成 THEN システムはJSON形式での構造化データを受け入れる
2. WHEN 開発者がYouTube動画IDを指定 THEN システムは対応するデータファイルを自動的に読み込む
3. WHEN 開発者が問題の時間設定を定義 THEN システムは開始時間・正解発表時間・終了時間を管理する
4. WHEN 開発者が複数の正解パターンを設定 THEN システムは配列形式での正解データを受け入れる
5. WHEN 開発者がデータ検証を実行 THEN システムは必須フィールドと時間データの妥当性をチェックする
